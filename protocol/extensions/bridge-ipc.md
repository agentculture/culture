# Bridge IPC contract — CC ↔ culture-bridge

Status: draft (Phase 2 of [the mesh rearchitecture plan](../../docs/superpowers/specs/2026-06-03-mesh-rearchitecture-plan.md))

The `culture-bridge` process exposes an AF_UNIX socket at a per-nick
path under `$XDG_RUNTIME_DIR/culture/<nick>.sock` (Linux) or
`~/Library/Caches/culture/run/<nick>.sock` (Darwin). The Claude Code
plugin connects on `SessionStart` and exchanges JSON-line-delimited
IPC messages with the bridge. Peer-uid is verified via
`SO_PEERCRED` / `getpeereid` before any byte is read; mismatched
uids are refused.

Two message shapes:

- **Request / response** (synchronous): CC sends `{type, id, …}`;
  bridge replies with `{id, ok, data?, error?, warnings?}`. The `id`
  is a UUID generated by CC.
- **Whisper** (push): bridge sends an unsolicited
  `{type: "whisper", whisper_type, message, …}` frame. Used for the
  bridge → CC push surface (inbound DMs, mentions, ROOMINVITEs,
  perm requests). CC is expected to dispatch on `whisper_type`.

Encoding: one JSON object per line, UTF-8, `\n`-terminated.

## Dropped verbs (versus the legacy claude/daemon.py IPC)

| Verb | Reason |
|---|---|
| `clear` | No SDK loop to clear. Workers' own daemons keep this verb. |
| `pause` | No SDK loop to pause. CC has its own sleep/resume via plugin tools. |
| `resume` | Symmetric with `pause`. |

## Whisper frame shape

```
{
  "type": "whisper",
  "whisper_type": "inbound_dm" | "inbound_mention" | "inbound_roominvite" | "perm_request",
  "message": "<JSON-encoded payload>",
  "id": "<bridge-generated UUID>",
  "ts": 1717440000.123
}
```

The `message` field carries the verb-specific payload (see table)
serialized as JSON. CC's IPC client is expected to JSON-decode it
when dispatching on `whisper_type`.

## Source of truth

The verb table is enforced by the dispatch dict in
[`culture/clients/bridge/daemon.py`](../../culture/clients/bridge/daemon.py)
(`_ipc_dispatch`). Any addition to this contract MUST update both
that dict and this document in the same PR.
