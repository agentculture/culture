## B. All-backends drift audit

CLAUDE.md mandates: "When adding or changing a feature in any agent
harness, propagate the change to all backends." Phase 4 introduces the
CC plugin and the bridge for **Claude only**. This is an intentional
carve-out — see Architectural Decision AD-3 in the rearch plan, and
the "All-backends carve-out" section of the v9.0.0-rc.1 CHANGELOG
entry.

| Change | claude | codex | copilot | acp | Documented carve-out? |
|---|---|---|---|---|---|
| Polling-prompt strip in SDK system prompt | ✅ done | ✅ done | ✅ done | ✅ done | n/a — applied uniformly |
| Bridge process + IPC + CC plugin | ✅ done | ❌ — autonomous daemon retained | ❌ — autonomous daemon retained | ❌ — autonomous daemon retained | YES — CHANGELOG "All-backends carve-out" section |
| Server-side DM spool | shared (in `agentirc/`) | shared | shared | shared | applies to all (server layer) |
| No-grant-ceiling broker | ✅ (shared `_perm_broker.py`) | ✅ | ✅ | ✅ | applies to all (shared code layer) |
| Worker auto-prefix | ✅ | ✅ | ✅ | ✅ | applies to all (cli/boss layer) |

**Drift class:** Boss-role bridge is Claude-only. **Documented** in the
v9.0.0-rc.1 CHANGELOG "All-backends carve-out" section, in the rearch
plan, and in this finding. A follow-up issue should track bringing
codex/copilot/acp bosses onto the bridge model in a later release —
listed in Task 7.6's follow-up list.
