### Note on `_on_welcome`

The task description flagged "_on_welcome's raw JOIN sends for pre-configured channels (separate site)" as a possible second patch site. Investigation confirms **no edit is needed there**:

`_on_welcome` iterates `self.channels` (pre-filled in `__init__` from the constructor's `channels=[...]` argument) and emits raw `JOIN <chan>` lines. Under the new confirmation model:

- If the server accepts the JOIN → it echoes back → `_on_join` sees the channel is already in `self.channels` and is a no-op. Correct.
- If the server rejects the JOIN with 474 → `_on_bannedfromchan` removes it from `self.channels`. Correct — failed startup-time JOINs no longer poison membership state.

So pre-configured channels remain in `self.channels` immediately at construction, but any that the server refuses are evicted on the 474. The optimistic-tracking bug only ever affected the runtime `join_channel` path (boss rejoining task channels after a fresh spawn), which is now fixed.
